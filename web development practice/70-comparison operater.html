Comparison operater

  1. Comparison operater are used in logical statements to determine equality or difference between variables or values. They will result in true or false only.
  2. == and === are used for equality check.
  3. == does not consider datatype of values being compared.
  4. == tries to convert one of the value and compare based on that.
  5. === also check data type of the values on the both side should be same otherwise it will reutrn false.
  6. For Example:
    • var a = 3;
    • console.log(a == 3);Return true.
    • console.log(a == "3");Return true.
    • console.log(a == 6); // Return False.
    • console.log(a === 3);Return true.
    • console.log(a === "3"); // Return False.
    • console.log(a === 6); // Return False.
    Opeater Description
    == Equal to.
    === Equal value and equal type.
    != not equal.
    !== Not equal value or not equal type.
    > Greater then.
    < Less then.
    >= Greater then and equal to.
    <= Less then and equal to.
  7. Examples:
    • var a = 3;
    • console.log(a == 6); // Return false
    • console.log(a === 6); // Return false
    • console.log(a != 6); // Return true
    • console.log(a !== 6); // Return true
    • console.log(a > 6); // Return false
    • console.log(a < 6); // Return true
    • console.log(a >= 6); // Return false
    • console.log(a <= 6); // Return true

Be Carefull.

  1. Javascript is a dynamic language and many decisions taken at start are still causing confusions.
  2. Comparing different ypes of values will result in answer which is not easily understandable.
  3. E.g: When comparing a string with a number, javascript will convert the string to a number while doing the comparison.
    An empty string converts to 0.
    A non-numeric string converts to NaN which is always false
