Statements

  1. A computer programme isa list of "instructions" to be"executed" by a computer.
  2. In a programming language, these programming instructions are called statements.
  3. A javascript programme is a list of statements.
  4. A statement can set a variable equal to value.
  5. A statement is also be a function call i.e document.write().
  6. Statement define that the script will do and how it will be done.
  7. Most javascript programmes contain many javascript statements.
  8. The statements ae executed, one by one, in the same order as they written.
  9. Each line below iss statement:
    • var a = 4;
      //Statement 01
    • var b = 2;
      //Statement 02
    • var c = 0;
      //Statement 03
    • c = a + b;
      //Statement 04
    • alert(a);
      //Statement 05
    • console.log(c);
      //Statement 06

End of statements with a semicolon;

  1. To end statement add a semicolon at the end of each executable statement.
  2. Semicolon seperates Javascript statements.
    • var a = 5;
  3. When Seperated by a semicolon, multiple statements on one line are allowed.LIKE THIS:
    • i = 3; j = 5; k = i + j;
    • alert(i); console.log(k);

End of statement without semicolon.

  1. Typically we end statements with a semicolon but in javascript semicolon is optional but is highly recomended to end a statement with a semicolon.
  2. The end of a statement is most often marked by pressing enter and starting a new line.
  3. For example
    • var a = 5//New line will end statement
    • alert(a)
    • a * 4
    • console.log(a)

      • This will make errors:

    • var a = 5 a * 4//Error, Will not work
    • alert(a) console.log(a)//Error, Will not work
